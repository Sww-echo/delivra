## Technical Approach Options

### Option A: Vue 3 + Vite + Vant (Recommended)

- 适合纯 H5 项目，路由、弹窗、表单、Picker、Tabs、列表等移动端组件成熟。
- 工程更轻，后续维护和自定义样式相对直接。
- 与现有打包产物中的 Vue Router 痕迹相匹配。

### Option B: uni-app H5

- 适合 H5 之外未来还要扩展小程序/App 的场景。
- 与现有打包产物中的 uni-app、uni-popup、uni-rate、z-paging 痕迹最接近。
- 框架约束更强，若只做 H5，复杂度高于 Vue + Vite。

### Option C: React + Vite + Ant Design Mobile

- React 生态成熟，Ant Design Mobile 覆盖常见移动端组件。
- 如果团队更熟 React 是可行选择，但与当前产物痕迹不如 Vue/uni-app 对齐。

## Decision (ADR-lite)

**Context**: 需要从零搭建移动端 H5 前端工程，需求文档覆盖大量页面跳转、弹窗、表单、列表和移动端控件。

**Decision**: 采用 Vue 3 + Vite + Vant。

**Consequences**: 纯 H5 场景下工程轻、开发快、组件覆盖度高；暂不优先考虑 uni-app 的多端能力，后续若要扩展小程序/App 需要重新评估。
